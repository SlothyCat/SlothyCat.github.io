# Security Controls

## Before we get started...
This is my first blog trying to document what I think and understand about the concepts that are taught in Security+. I hope that this becomes a consistent journey where I document down an improving version of myself.

## Quick Summary
Security Controls are mechanisms that we can use to **prevent**, **detect** and **recover** from security events/attacks. They do not need to be technical using programming at all!


The controls are split into various **categories** and **types**:
- Category: what it is made of
- Type: what it does

---

### The 4 Categories

#### Technical
Implemented using **technology**, which was what I thought security control was all about. (It is only the first category 🤯)

> Examples: OS policies, firewalls, antivirus, software controls

#### Managerial
Implemented using **policies and procedures**

> Examples: Onboarding Policy, Compliance Documentation, Standard Operating Procedures

#### Operational
Implemented by **People**

> Examples: Security Guards, Receotion, Security Training

#### Physical
Limits **physical access** to buildings, rooms or devices

> Examples: Guard Shacks, Fences, Locks, Badge Readers

---

### The 6 Control Types

#### Preventive
Stops the illegal access before it even happens.


#### Deterrent
Discourages Attackers but do not actually physically stop them. It makes them think twice before acting.

#### Detective
Identifies and Logs breaches when they *have occured or during* but do not prevent it.

#### Corrective
Applied after an event is detected. This aims to reverse the impact of the attack and/or restore operations.

#### Compensating
Alternative control that is used when the ideal control is not available. It is often temporary and as a fix when ideal control is not yet feasible.

#### Directive
Directs people towards a more secure behaviour. It is the weakest control type as it relies on user compliance. 

It is hard to get people to follow guidelines when there are more convenient paths.

## More Examples: Category and Type

| Control Type     | Technical               | Managerial               | Operational          | Physical                       |
|------------------|-------------------------|--------------------------|----------------------|--------------------------------|
| **Preventive**   | Firewall rule           | Onboarding policy        | Guard shack          | Door lock                      |
| **Deterrent**    | Splash screen           | Demotion threat          | Reception desk       | Warning sign                   |
| **Detective**    | System logs             | Login report review      | Property patrol      | Motion detector                |
| **Corrective**   | Restore from backup     | Incident reporting policy| Contact law enforcement | Fire extinguisher           |
| **Compensating** | Firewall rule (temp patch) | Separation of duties  | Multiple guards on duty | Power generator             |
| **Directive**    | File storage policy     | Compliance policy        | Security training    | "Authorised Personnel Only" sign |

---

### Thoughts:
There are multiple aspects of Security Control and they come in at different parts of the attack phase.

- **Timing**: Deterrent → Preventive → Detective → Corrective
- **Nature**: Compensating (fills a gap) and Directive (guides behaviour) sit orthogonally to timing
